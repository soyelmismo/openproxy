## 2024-05-24 - SSE Streaming Error Leak Vulnerability: [API keys in SSE stream] Learning: [Streaming handlers (SSE) didn't apply the same truncation/redaction as standard JSON error responses, allowing upstream proxy keys or verbose stack traces to leak to clients if a connection dropped or errored.] Prevention: [Always pipe error messages through `redact_error_msg` and `truncate_error_message` before constructing SSE data frames.]
## 2025-02-18 - [SQL Injection via unescaped string in PRAGMA] Vulnerability: [SQLite PRAGMA statements typically do not support parameterized queries in rustqlite, making string concatenation dangerous] Learning: [Always escape single quotes via `replace("'", "''")` when building PRAGMA queries with format!] Prevention: [Review all SQL queries for string format calls or prefer prepared statements where supported]
## 2025-02-18 - [Denial of Service via unwrap] Vulnerability: [Unhandled panics on JSON serialization or axum response builder] Learning: [Uncontrolled user input can crash the application via explicit unwrap()] Prevention: [Replace unwrap() with appropriate error handling variants or unwrap_or_else/unwrap_or_default providing safe fallback responses]
## 2025-02-18 - [Information Disclosure] Vulnerability: [Admin test API exposed unredacted error messages from upstream connection failures] Learning: [Even authenticated endpoints should securely redact secrets returned by upstreams, as HTTP body parsing might inadvertently leak API keys] Prevention: [Ensure all explicit error.to_string() calls that cross boundaries are sanitized through a global `redact_error_msg` filter]
